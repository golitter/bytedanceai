# 契约层（Contract Layer）使用指南

## 概述

`contracts/` 目录是三端（Frontend / Backend / AgentEnd）共享协议的**单一来源 (Single Source of Truth)**。

## 目录结构

```
contracts/
├── schemas/          # YAML 格式的 JSON Schema 契约定义
│   ├── event-types.yaml     # EventType 枚举 + StreamEvent 结构
│   ├── agent-request.yaml   # AgentType 枚举 + AgentRequest 结构
│   ├── agent-response.yaml  # AgentResponse 结构
│   ├── session-state.yaml   # SessionState 枚举 + 合法转换表
│   ├── message.yaml         # 持久化消息（role: user/agent + status: streaming/completed/failed + Redis Stream 追踪）
│   └── validate-repo-path.yaml # Repo 路径验证协议（请求/响应结构）
├── logs/             # 契约变更审计记录（YYYY-MM-DD-<description>.md）
└── AGENTS.md         # 契约层说明
```

## 代码生成

运行 `make generate`，从 `contracts/schemas/*.yaml` 自动生成三端类型文件：

| Schema | Python | TypeScript | Go |
|--------|--------|------------|-----|
| event-types.yaml | `agentend/src/generated/events.py` | `frontend/src/generated/events.ts` | `backend/internal/generated/events.go` |
| agent-request.yaml | `agentend/src/generated/request.py` | `frontend/src/generated/request.ts` | `backend/internal/generated/request.go` |
| agent-response.yaml | `agentend/src/generated/response.py` | `frontend/src/generated/response.ts` | `backend/internal/generated/response.go` |
| session-state.yaml | `agentend/src/generated/session.py` | `frontend/src/generated/session.ts` | `backend/internal/generated/session.go` |

每个生成文件首行包含 `# Code generated by make generate. DO NOT EDIT.`，手动修改会被下次生成覆盖。

> **注意**：`message.yaml` 和 `validate-repo-path.yaml` 尚未纳入 `make generate` 代码生成流程，各端目前直接引用 Pydantic/GORM/TS 类型定义。

## 契约变更流程

1. 修改 `contracts/schemas/*.yaml` 中的协议定义
2. 运行 `make generate` 生成三端类型
3. 在 `contracts/logs/` 写入变更记录
4. 提交时 pre-commit hook 会自动检测并验证一致性

## Pre-commit Hook

`.husky/pre-commit` 包含三个阶段：

1. **lint-staged** — 代码格式检查
2. **契约变更检测** — 检测 staged 文件是否涉及契约相关目录，触发时自动运行 `make generate` 并暂存
3. **Secret 验证** — 从 `/dev/tty` 读取用户输入，与 `.husky/.commit-secret` 对比

### 契约相关目录

以下目录的文件变更会触发契约检测：

- `agentend/src/schemas/`
- `agentend/src/api/`
- `agentend/src/adapters/`
- `backend/internal/model/`、`handler/`、`types/`
- `frontend/src/types/`、`src/api/`
- `contracts/`

### Secret 配置

```bash
echo 'your-secret' > .husky/.commit-secret
```

该文件已在 `.gitignore` 中，不会被提交。
