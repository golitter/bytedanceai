# AgentHub Visual Style Guide

> Dark Utilitarian — 深 色 · 平 坦 · 克 制 · 密 集

## 定位

AgentHub 不是 AI 聊天软件，而是 AI Runtime Workspace。用户在这里调度 Agent、看 Diff、看 Logs、看 Preview、看多 Agent 协作流。视觉风格必须传递"精密控制台"的气质，而非"聊天机器人"。

参考产品气质（不是照搬 UI）：

| 产品 | 借鉴什么 |
|------|---------|
| Linear | 克制暗色、精确间距、极简交互 |
| Cursor | IDE 氛围、代码为核心 |
| Vercel | 部署状态可视化、干净面板 |
| Slack | 对话流的 IM 节奏感 |
| GitHub Actions | Event Timeline 的运行时感 |

## 一、色彩体系

### 1.1 背景色阶

层级靠色阶区分，不靠阴影和模糊。背景之间差值约 10-15 个 RGB 单位，足够分辨但不突兀。

| 用途 | 色值 | 变量名 |
|------|------|--------|
| 主画布（中栏聊天区） | `#0A0B0E` | `--bg-canvas` |
| 侧边栏 | `#111318` | `--bg-sidebar` |
| 卡片/面板 | `#1A1D24` | `--bg-card` |
| 悬停态/提升态 | `#22262F` | `--bg-hover` |
| 按下态/最浅提升 | `#2C313B` | `--bg-active` |

### 1.2 文字色阶

| 用途 | 色值 | 变量名 |
|------|------|--------|
| 主要文字（标题、正文） | `#E8EBF0` | `--text-primary` |
| 次要文字（时间戳、描述） | `#8B91A0` | `--text-secondary` |
| 辅助文字（占位符、禁用态） | `#5A6070` | `--text-tertiary` |

### 1.3 品牌色

全局只有一个品牌色，极度克制地使用——仅出现在选中态、发送按钮、当前会话标记、焦点环。

| 用途 | 色值 | 变量名 |
|------|------|--------|
| 品牌主色 | `#6366F1` | `--color-brand` |

### 1.4 Agent 标识色

不算装饰色，是功能色——用于区分 Agent 身份。只用在 Agent 头像和消息左侧色条。

| Agent | 色值 | 变量名 |
|-------|------|--------|
| Claude Code | `#DA7756` | `--agent-claude` |
| OpenCode | `#10B981` | `--agent-opencode` |
| Orchestrator | `#EAB308` | `--agent-orchestrator` |
| Codex | `#6366F1` | `--agent-codex` |

### 1.5 语义色

| 状态 | 色值 | 变量名 |
|------|------|--------|
| 成功 | `#22C55E` | `--color-success` |
| 警告 | `#F59E0B` | `--color-warning` |
| 错误 | `#EF4444` | `--color-error`（即 `--destructive`） |

### 1.6 色彩禁令

- 不使用任何渐变（`linear-gradient`、`radial-gradient`）
- 不使用 `backdrop-blur`（不做毛玻璃效果）
- 不使用纯白文字（`#FFFFFF`），正文用 `#E8EBF0`
- 不使用纯黑文字（`#000000`）
- 不在 UI 框架上使用 Agent 标识色，它们只标识身份

## 二、字体系统

### 2.1 字体选择

| 用途 | 字体 | 理由 |
|------|------|------|
| UI 文字 | Geist Sans | Vercel 出品，为开发者工具设计；比 Inter 更紧凑，适合高密度界面 |
| 代码文字 | Geist Mono | 与 Geist Sans 配套，UI 文字和代码视觉统一 |

### 2.2 字号体系

| 用途 | 字号 | 字重 | 行高 | 场景 |
|------|------|------|------|------|
| 页面标题 | 20px | 600 (SemiBold) | 1.3 | Task 名称 |
| 区块标题 | 14px | 600 | 1.4 | 面板标题、卡片标题 |
| 正文 | 14px | 400 (Regular) | 1.6 | 聊天消息、描述文字 |
| 辅助文字 | 12px | 400 | 1.5 | 时间戳、标签、次要信息 |
| 代码 | 13px | 400 | 1.65 | 代码块、终端输出 |
| 微标注 | 11px | 500 (Medium) | 1.4 | Badge、Agent 状态标签 |

### 2.3 字间距

- UI 文字：`-0.01em`（略收紧，增加密度感）
- 代码文字：`0`（等宽默认间距）
- 大写标签（Badge）：`+0.05em`（拉开间距，提升可读性）

## 三、圆角

| 元素 | 圆角 | 说明 |
|------|------|------|
| 按钮 | 6px | 方正、工具感 |
| 输入框 | 8px | 内敛 |
| 卡片 | 10px | 稍圆润但不圆滑 |
| 面板 | 12px | 最大圆角，仅用于大面板 |
| Agent 头像 | 8px | 圆角方形，区别于用户圆形头像 |
| 用户头像 | 50% | 完全圆形 |
| Badge/标签 | 9999px | 胶囊形 |

### 圆角禁令

- 不使用 16px 以上的大圆角
- 不使用 `rounded-full` 用在按钮或卡片上
- 不做 iOS 风格的大圆角

---

## 四、边框

- 通用边框：`1px solid rgba(255, 255, 255, 0.06)`
- 分割线：`1px solid rgba(255, 255, 255, 0.04)`（比边框更淡）
- 焦点环：`2px solid var(--color-brand)` + `2px offset`
- 不用实色边框（如 `#333`）

---

## 五、阴影与深度

### 核心原则：几乎不用阴影

深度靠背景色差表达，不靠 `box-shadow`。

| 层级 | 色值 | 场景 |
|------|------|------|
| 层级 0（最底） | `#0A0B0E` | 画布 |
| 层级 1 | `#111318` | 侧边栏 |
| 层级 2 | `#1A1D24` | 卡片 |
| 层级 3（最高） | `#22262F` | 悬浮卡片 |

唯一允许阴影的场景——弹出菜单/下拉框（需要明确脱离页面）：

```css
box-shadow: 0 4px 24px rgba(0, 0, 0, 0.4);
```

## 六、间距系统

基础单位：**4px**

### 间距档位

| 名称 | 值 | 场景 |
|------|-----|------|
| xs | 4px | 图标与文字间距、内联元素间距 |
| sm | 8px | 紧凑元素间距 |
| md | 12px | 列表项间距 |
| lg | 16px | 区块内间距（卡片 padding） |
| xl | 24px | 区块间间距 |
| 2xl | 32px | 大区块间距 |
| 3xl | 48px | 页面级间距（极少使用） |

### 组件内部 padding

| 组件 | padding |
|------|---------|
| 按钮（小） | 8px 16px |
| 按钮（标准） | 10px 20px |
| 卡片 | 16px |
| 输入框 | 10px 14px |
| 面板 | 16-20px |
| 列表项 | 10px 12px |

### 密度原则

这是开发者工具，间距紧凑。以 8-16px 为主。参考 Linear 的密度，不参考 Apple 的呼吸感。

## 七、图标

选择：**Lucide React**（通过 `strokeWidth={1.25}` 模拟细线视觉）

理由：
- 与 shadcn/ui 深度集成，组件库内置 Lucide
- 生态成熟，图标覆盖面广
- 通过 `strokeWidth={1.25}` 可模拟 Phosphor Light 的细线风格，兼顾精致感
- 与 development-strategy.md 保持一致，避免文档矛盾

不选：
- FontAwesome — 太重，不适合精致 UI
- Material Icons — 圆润风格不匹配

## 八、动效

### 哲学

存在感极弱，几乎感觉不到，但没有就会觉得"卡"。

### 时长

| 场景 | 时长 | 缓动 |
|------|------|------|
| 微交互（hover、focus） | 120ms | ease-out |
| 状态切换（展开/折叠） | 200ms | ease-out |
| 面板出入（右侧面板滑入） | 250ms | ease-out |
| 页面级过渡 | 300ms | ease-out |

### 允许的动效

- hover 背景色变化（120ms ease-out）
- 面板滑入/滑出（250ms ease-out，`transform: translateX`）
- Agent 思考指示器（三点跳动）
- 状态灯颜色变化（200ms ease-out）
- 新消息入场（底部滑入，300ms ease-out，轻微透明度渐变）

### 禁止的动效

- 毛玻璃/模糊动画
- 渐变流光
- 粒子效果
- 3D 旋转
- 大幅位移动画
- 弹簧物理动画（overshoot/bounce）
- 骨架屏闪光扫描效果

### 性能约束

- 只动画 `transform` 和 `opacity`，不动画 `top`、`left`、`width`、`height`
- `backdrop-blur` 只用于固定/粘性元素（导航栏、遮罩层），不用在滚动容器上

## 九、Agent 身份系统

每个 Agent 拥有统一的视觉标识格式：

```
身份 = 图标 + 标识色 + 头像形状 + 状态灯
```

### 头像

- 形状：圆角方形（8px 圆角）— 区别于用户的圆形头像，暗示"这是工具/服务"
- 尺寸：32px（消息流中）、24px（侧边栏紧凑模式）
- 背景：Agent 标识色，带 8px 同色模糊光晕（这是唯一允许的"发光"效果，仅头像使用）

### 状态灯

头像右下角 4px 圆点：

| 状态 | 颜色 | 动效 |
|------|------|------|
| 就绪 | 绿色 `#22C55E` | 脉冲（2s 循环，透明度 0.6→1→0.6） |
| 执行中 | 黄色 `#F59E0B` | 旋转（1.5s 循环） |
| 离线 | 灰色 `#5A6070` | 无 |
| 出错 | 红色 `#EF4444` | 无 |

### 消息色条

Agent 消息左侧带 3px 细条，颜色为该 Agent 标识色，用于在群聊中快速区分发言人。

## 十、组件视觉规范

### 10.1 消息气泡

**用户消息**：
- 右对齐
- 背景：`rgba(99, 102, 241, 0.08)`（品牌色极低透明度）
- 边框：`1px solid rgba(99, 102, 241, 0.15)`
- 圆角：10px

**Agent 消息**：
- 左对齐
- 背景：`var(--bg-card)` `#1A1D24`
- 左侧色条：3px，Agent 标识色
- 圆角：10px

**系统消息**（Orchestrator 调度、状态变更）：
- 居中
- 文字：`var(--text-secondary)` `#8B91A0`
- 字号：12px
- 无气泡背景

### 10.2 代码块

- 背景：`#0D0F14`（比画布更深一级）
- 字体：Geist Mono，13px
- 行号：左侧独立列，`var(--text-tertiary)` `#5A6070`
- 语法高亮：Tokyo Night 主题色系
- 圆角：8px
- 内边距：16px

### 10.3 Diff 卡片

- 旧代码区域：`rgba(239, 68, 68, 0.1)` 红色调背景
- 新代码区域：`rgba(34, 197, 94, 0.1)` 绿色调背景
- 删除行左侧 `-` 标记，新增行左侧 `+` 标记
- 顶部标签栏：`--- 旧版本` / `+++ 新版本`
- 底部操作栏：品牌色按钮"应用此 Diff"

### 10.4 输入区

- 背景：`var(--bg-card)` `#1A1D24`
- 边框：`1px solid rgba(255, 255, 255, 0.06)`
- 焦点态边框：`1px solid var(--color-brand)`
- 圆角：8px
- 最小高度：48px，支持多行展开
- 占位符文字：`var(--text-tertiary)` `#5A6070`

### 10.5 侧边栏列表项

- 默认背景：透明
- Hover 背景：`var(--bg-hover)` `#22262F`
- 选中态：左侧 2px 品牌色竖线 + `var(--bg-hover)` 背景
- 内边距：10px 12px

## 十一、总结

```
Dark Utilitarian — 三原则

1. 层级靠色阶，不靠阴影
   背景 5 级灰阶，从 #0A0B0E 到 #2C313B

2. 色彩靠功能，不靠装饰
   全局 1 个品牌色 + 3 个 Agent 色 + 4 个语义色

3. 动效靠存在，不靠表现
   120-300ms ease-out，只做状态过渡
```

它看起来像 Linear 的暗色面板，读起来像 JetBrains IDE 的代码区，用起来像 Slack 的对话流。它是一个安静的、精密的控制台。
