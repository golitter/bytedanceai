## ADDED Requirements

### Requirement: contracts/ 目录结构
系统 SHALL 在项目根目录维护 `contracts/` 目录作为三端共享契约的单一来源。目录结构 SHALL 包含 `schemas/`（JSON Schema 定义）和 `logs/`（变更审计记录）。

#### Scenario: contracts/ 目录初始化
- **WHEN** 项目首次设置契约层
- **THEN** SHALL 创建 `contracts/schemas/` 和 `contracts/logs/` 目录，`schemas/` 中包含所有跨端协议的 JSON Schema 文件

### Requirement: JSON Schema 契约格式
每个跨端协议定义 SHALL 以 JSON Schema 文件存储在 `contracts/schemas/` 目录中。文件名 SHALL 使用 kebab-case（如 `agent-request.yaml`）。`agent-request.yaml` 中的 `AgentType` 枚举 SHALL 包含 `claude-code`、`opencode`、`orchestrator`、`codex` 四个值。

#### Scenario: 有效的 JSON Schema 文件
- **WHEN** 定义 EventType 契约
- **THEN** SHALL 创建 `contracts/schemas/event-types.json`，包含所有事件类型枚举值作为 schema 的 enum 约束

#### Scenario: Schema 文件命名规范
- **WHEN** 添加新的跨端协议定义
- **THEN** 文件名 SHALL 为 kebab-case 格式的 `.json` 文件，放置在 `contracts/schemas/` 目录下

#### Scenario: AgentType 包含 codex 枚举值
- **WHEN** 查看 `contracts/schemas/agent-request.yaml` 的 AgentType 枚举定义
- **THEN** SHALL 包含枚举值 `codex`，描述为 "Codex CLI"

#### Scenario: 三端生成类型包含 codex
- **WHEN** 运行 `make generate`
- **THEN** 三端的 `generated/request.*` 文件中 AgentType 枚举 SHALL 包含 `codex` 值

### Requirement: 代码生成管线
系统 SHALL 提供 `make generate` 命令，从 `contracts/schemas/` 中的 JSON Schema 文件自动生成三端类型文件。生成代码 SHALL 放置在各端的 `generated/` 目录中。

#### Scenario: 生成 Python 类型
- **WHEN** 运行 `make generate`
- **THEN** SHALL 从 `contracts/schemas/event-types.json` 生成 `agentend/src/generated/events.py`，包含与 schema 一致的 Pydantic 模型和 Enum

#### Scenario: 生成 TypeScript 类型
- **WHEN** 运行 `make generate`
- **THEN** SHALL 从 `contracts/schemas/event-types.json` 生成 `frontend/src/generated/events.ts`，包含与 schema 一致的 TypeScript type 定义

#### Scenario: 生成 Go 类型
- **WHEN** 运行 `make generate`
- **THEN** SHALL 从 `contracts/schemas/event-types.json` 生成 `backend/internal/generated/events.go`，包含与 schema 一致的 Go 类型定义

### Requirement: 生成代码不可手改
`generated/` 目录下的文件 SHALL 完全由 `make generate` 生成，包含自动生成头注释。人工修改生成文件 SHALL 在下次 `make generate` 时被覆盖。

#### Scenario: 生成文件包含头注释
- **WHEN** 运行 `make generate`
- **THEN** 每个生成文件 SHALL 在首行包含 `// Code generated by make generate. DO NOT EDIT.` 或等效标记

#### Scenario: 手动修改生成文件被覆盖
- **WHEN** 手动修改 `generated/` 下的文件后运行 `make generate`
- **THEN** 修改 SHALL 被重新生成的文件覆盖

### Requirement: 三端导入生成类型
三端代码 SHALL 从各自 `generated/` 目录导入跨端协议类型，SHALL NOT 在端代码中手写重复的类型定义。

#### Scenario: AgentEnd 使用生成类型
- **WHEN** AgentEnd 代码需要使用 EventType 枚举
- **THEN** SHALL 从 `src/generated/events.py` 导入，SHALL NOT 在 `src/schemas/` 中重复定义

#### Scenario: Frontend 使用生成类型
- **WHEN** Frontend 代码需要使用事件类型
- **THEN** SHALL 从 `src/generated/events.ts` 导入

#### Scenario: Backend 使用生成类型
- **WHEN** Backend 代码需要使用事件类型
- **THEN** SHALL 从 `internal/generated/events.go` 导入
